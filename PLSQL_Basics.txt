Rules to be followed for naming a varaible
--------------------------------------------
1. Every variable should start with alphabet, from second char
   onwards it can either variable or numeric.
   name  -->valid
   1name --- invalid
   name1 --valid
   
 
2. It will not allow blank spaces
   first name ---> invalid
   firstName ---> valid
   first_name  --> valid

3. It cannot have any special operators or special characters
   except underscore  (_)
   first+name  -- invalid
    last*  ---invalid
   first^name --invalid

4. It cannot be a keyword
   varchar  , number, dbms,output
   
Conventions
------------
1. every variable name must specify its purpose
2. whenever your variable need to be with two word combination
   we have to use either _ or camel case
   first_name   // firstName
   last_name    // lastName
3. if any variable value is not changing frequently better to 
   define it as constant.


The variable applicable on variable will also be applicable 
for constant, procedure, function, trigger or any other
valid sql object
